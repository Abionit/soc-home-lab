# Executive Report

## Summary

This lab demonstrates a compact SOC detection workflow using Wazuh and an analytics layer for alert review, triage, and reporting.

## Business Value

The project shows how raw security events can be transformed into operational insight:

- which detections are firing
- which assets and users appear most often
- which alerts should be triaged first
- which detections require tuning
- how SOC activity can be communicated clearly

## Key Capabilities Demonstrated

- Wazuh SIEM/XDR deployment planning
- Custom detection rule design
- MITRE ATT&CK mapping
- Alert triage prioritization
- SOC KPI reporting
- Python-based analytics over alert data
- Executive and technical documentation

## Metrics Produced

- total alerts
- critical and high severity alerts
- detections by rule
- alerts by host and user
- active triage queue
- MITRE technique coverage
- severity distribution

## Portfolio Positioning

This project supports security analytics, SOC analyst, junior detection engineering, and data analyst roles where operational reporting and alert analysis matter.
