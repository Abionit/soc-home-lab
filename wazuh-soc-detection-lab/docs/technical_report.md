# Technical Report

## Detection Engineering Scope

The lab includes custom rules for authentication attacks, suspicious command execution, privilege changes, file integrity events, malware test artifacts, web attack patterns, and credential access indicators.

## Rule Strategy

Custom rules use IDs in the `100000` to `120000` range, aligned with Wazuh custom rule guidance.

Each rule includes:

- unique rule ID
- severity level
- readable description
- MITRE ATT&CK technique
- detection group

## Validation Strategy

Validation uses controlled synthetic log events generated by `scripts/generate_lab_events.py`. These events are designed to trigger the custom rule patterns without requiring harmful tools or real attacks.

## Analytics Strategy

Alert data is processed with Python into:

- CSV metrics
- triage queues
- detection summaries
- SQLite tables
- Markdown executive report

This adds a data analysis layer on top of the SOC workflow.

## Tuning Considerations

- Brute-force thresholds should be adapted to expected authentication volume.
- Encoded PowerShell alerts may require allowlists for administrative automation.
- FIM alerts should prioritize sensitive paths and reduce noisy directories.
- Web attack detections should be correlated with HTTP status codes and source IP frequency.
- Credential access indicators should be handled as high-priority alerts even in lab mode.
